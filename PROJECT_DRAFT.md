# 과제 초안

## 제목
대학원생용 연구 보조 GPT: 관심 키워드·기존 독서 이력·저명성·난이도를 반영한 논문 추천 및 읽기 가이드 에이전트

## 1. 문제 정의
논문 스터디를 진행하면서 가장 큰 어려움 중 하나는 “지금 내게 꼭 필요한 논문”을 고르는 일이다. 단순 검색은 결과가 너무 많고, 인용 수만 보면 최신성을 놓치며, 생성형 AI에게 막연히 질문하면 그럴듯하지만 근거가 불명확한 추천이 나오기 쉽다. 특히 대학원생 입장에서는 관심 키워드, 기존에 읽은 논문, 현재 이해 수준, 논문의 저명성까지 함께 고려한 추천이 필요하다.

본 프로젝트는 이러한 문제를 해결하기 위해, 사용자의 관심 주제와 이전 독서 이력, 난이도 선호를 입력받아 적절한 논문을 추천하고, 사용자가 선택한 논문에 대해 실제 스터디에 도움이 되는 읽기 방향성을 제시하는 연구 보조 GPT를 제안한다.

## 2. 과제 주제와의 부합성
본 프로젝트는 과제 설명의 선택 주제 중 다음과 직접 부합한다.

- 주제 2: **LLM을 활용한 (전공) 연구 보조 파이프라인 구축 및 평가**
- 주제 3: **전공 특화 GPTs 개발 프로젝트**

즉, 단순 챗봇 제작이 아니라 실제 연구 활동 보조를 위한 파이프라인과 업무 특화 GPT라는 점에서 과제 취지와 맞는다.

## 3. 강의 내용과의 연계
### 3-1. Prompt Engineering
수업에서는 좋은 결과를 얻기 위해 목적성, 맥락 제공, 구체성/명확성, 예시 제공, 구조화가 중요하다고 다루었다. 본 시스템은 이를 그대로 반영한다.

- 목적성: must-read 논문 추천이라는 목표를 명확히 설정
- 맥락 제공: 관심 키워드, 읽은 논문, 난이도, 최신/고전 선호를 함께 전달
- 구체성: “좋은 논문 추천”이 아니라 “지금 이 사용자에게 맞는 논문 추천”으로 문제를 좁힘
- 예시 제공: JSON 형태의 추천 예시를 few-shot으로 포함
- 구조화: 추천 결과를 고정된 필드(JSON)로 반환하도록 설계

### 3-2. LLM
수업에서는 LLM이 next-token prediction 기반으로 문장을 생성하며, context window의 제약을 받는다고 설명했다. 본 시스템은 무작정 긴 후보군을 모두 던지지 않고, 먼저 heuristic scoring으로 shortlist를 만든 뒤 LLM에게 전달한다. 즉, context window와 비용을 고려한 설계다.

### 3-3. AI Design & Control
수업에서는 모델 성능과 시스템 성능이 다르며, 입력-맥락-출력-검증 루프가 필요하다고 강조했다. 본 시스템 역시 다음과 같이 설계했다.

- 입력 정규화: query / history / difficulty / freshness 분리
- 출력 통제: JSON only
- 검증: JSON 파싱 실패 시 fallback
- 사람 개입: 최종 논문 선택은 사용자가 수행
- 추적 가능성: 추천 결과와 점수를 run 파일로 저장

따라서 이 프로젝트는 “프롬프트만 잘 쓰는 것”이 아니라, 실제 서비스형 AI 설계 관점을 반영한 사례다.

### 3-4. Responsible AI
연구 보조 도구에서 가장 위험한 문제는 환각이다. 본 시스템은 메타데이터 기반 추천을 원칙으로 하며, 논문 본문에 없는 정보는 모른다고 명시하게 설계했다. 또한 로그를 남겨 추천 과정의 추적 가능성을 확보했다.

## 4. 시스템 설계
### 4-1. 전체 파이프라인
사용자 입력 → query expansion → 논문 후보 수집(Semantic Scholar / OpenAlex / arXiv) → 후보 병합 및 중복 제거 → 점수화 → LLM 기반 재정렬 및 추천 사유 생성 → 사용자가 논문 선택 → 읽기 가이드 생성

### 4-2. 점수화 기준
- 관련성: 관심 키워드와 제목/초록/분야의 일치도
- 저명성: citation_count, influential_citation_count, citation velocity
- 난이도 적합도: beginner/intermediate/advanced와의 차이
- 새로움: 이전 독서 이력과의 중복 정도
- 최신성: classic / balanced / latest 선호 반영

## 5. 구현
Python 기반으로 CLI 형태로 구현했다.

- `app.py`: 실행 진입점
- `scholarly_clients.py`: Semantic Scholar, OpenAlex, arXiv 검색
- `scorer.py`: 후보 점수화
- `prompts.py`: 구조화된 프롬프트
- `llm.py`: OpenAI Responses API 연동
- `agent.py`: 전체 파이프라인 orchestration

실행 결과는 JSON과 Markdown으로 저장되므로 GitHub나 Notion 정리에 바로 활용할 수 있다.

## 6. 기대 효과
- 논문 탐색 시간 단축
- 현재 수준에 맞는 논문 선택 가능
- 단순 추천이 아니라 “왜 읽어야 하는가”와 “어떻게 읽어야 하는가”까지 제공
- 연구 스터디의 진입장벽 완화

## 7. 한계
- 본문 PDF를 직접 읽지 않으므로 세부 실험 내용까지는 분석하지 못함
- 난이도 추정은 heuristic이라 오차 가능
- 인용 수 기반 저명성 판단은 신생 분야에 불리할 수 있음
- LLM 없이 사용하면 자연스러운 설명 품질이 낮아질 수 있음

## 8. 개선 방향
- PDF 본문 업로드 및 section-level 분석
- citation graph 기반 읽는 순서 자동 제안
- 분야별 난이도 예측 모델 추가
- Streamlit UI 구현
- 사용자 피드백 기반 개인화 강화

## 9. 결론
본 프로젝트는 대학원생의 실제 논문 스터디 문제를 해결하기 위한 연구 보조 GPT이며, Prompt Engineering, LLM, AI Design & Control, Responsible AI를 자연스럽게 반영한 과제 주제다. 또한 단순 아이디어 제안에 그치지 않고 VS Code에서 실행 가능한 형태로 구현했다는 점에서 과제의 결과물 완성도 측면에서도 의미가 있다.
